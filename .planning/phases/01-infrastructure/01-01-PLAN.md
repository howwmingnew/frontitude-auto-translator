**proxy/src/index.js** -- The Worker entry point (~70-80 lines). Implement:

1. `export default { async fetch(request, env) }` -- main handler
2. **OPTIONS handling:** If `request.method === 'OPTIONS'`, call `handleOptions(request, env)`. Validate origin first. Return preflight response with:
   - `Access-Control-Allow-Origin: {env.ALLOWED_ORIGIN}`
   - `Access-Control-Allow-Methods: GET, OPTIONS`
   - `Access-Control-Allow-Headers: Authorization, Content-Type`
   - `Access-Control-Max-Age: 86400`
3. **Method check:** Only allow GET. Return `405 Method not allowed` for anything else.
4. **Origin validation:** Read `request.headers.get('Origin')`. If it does not match `env.ALLOWED_ORIGIN`, return `403 Forbidden`.
5. **Path parsing:** Extract pathname. Must start with `/bitbucket/`. Strip that prefix to get `bbPath`. Build target URL: `https://api.bitbucket.org/{bbPath}{search}`.
6. **Path whitelist via `isAllowedPath(path)` function:** Allow only:
   - `2.0/workspaces/{ws}/search/code` (code search)
   - `2.0/repositories/{ws}/{repo}` (connection test -- exact match or with trailing path)
   - `2.0/repositories/{ws}/{repo}/src/{rest}` (file content)
   - Regex: `/^2\.0\/(workspaces\/[^/]+\/search\/code|repositories\/[^/]+\/[^/]+(\/src\/.+)?)(\?.*)?$/`
   - Note: The regex matches the path portion only (no query string). Query string is passed through separately via `url.search`.
7. **Forward request:** `fetch(bbUrl, { headers: { 'Authorization': 'Bearer ' + env.BITBUCKET_TOKEN, 'Accept': 'application/json' } })`
8. **Response:** Clone response, set `Access-Control-Allow-Origin` and `Vary: Origin` headers, return.
9. **Error responses:** Use `new Response(body, { status, headers: corsHeaders(env) })` pattern so error responses also have CORS headers (browser can read error messages).

Helper function `corsHeaders(env)`:
```javascript
function corsHeaders(env) {
  return {
    'Access-Control-Allow-Origin': env.ALLOWED_ORIGIN,
    'Vary': 'Origin'
  };
}
```
