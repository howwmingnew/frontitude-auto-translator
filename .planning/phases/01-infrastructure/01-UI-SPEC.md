## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (0.25rem) | Inline badge padding-block, icon gaps |
| sm | 8px (0.5rem) | Compact element spacing, gap between badge and test button |
| md | 12px (0.75rem) | Input padding, card internal row spacing |
| lg | 16px (1rem) | Error message margin, section margins within drawer |
| xl | 24px (1.5rem) | Card padding (existing `.card` uses 1.5rem) |
| 2xl | 32px (2rem) | Container padding, upload section spacing |

Exceptions:
- Input padding uses 8px vertical / 12px horizontal (0.5rem / 0.75rem) matching existing `select` and `input` patterns.
- `.btn-test-key` uses 6px / 12px (0.375rem / 0.75rem) matching existing test button padding.

**Source:** Extracted from existing CSS patterns in `index.html` lines 251, 334, 341, 360, 375, 706-711.

## Interaction States

### Test Connection Button

| State | Visual | Trigger |
|-------|--------|---------|
| Disabled | opacity 0.4, cursor not-allowed | Any required field (proxy URL, workspace, repo, token) is empty |
| Enabled | Default `.btn-test-key` style | All required fields populated |
| Loading | Text changes to "Connecting..." (`bbConnecting`), disabled | User clicks Test Connection |
| Success | `.test-key-success` class (green bg/border/text), text "Connected" | API returns 200 |
| Error | Reverts to enabled state, error toast shown | API returns non-200 or network error |

### Connection Badge

| State | Class | Text | Color |
|-------|-------|------|-------|
| Not connected | `badge-none` | "Not Connected" | `--badge-none-bg` / `--badge-none-text` |
| Connecting | `badge-none` | "Connecting..." | `--badge-none-bg` / `--badge-none-text` |
| Connected | Custom: `badge-connected` | "Connected" | `--success-light` bg / `--success` text (matches `.test-key-success` pattern) |
| Failed | Custom: `badge-error` | "Not Connected" | `--error-light` bg / `--error` text |

### Input Fields

| State | Visual |
|-------|--------|
| Empty | Placeholder text in `--text-muted` color |
| Focused | `box-shadow: var(--focus-ring)`, border-color transitions to `--primary-border` |
| Filled | Normal text color `--text` |
| Token field | `type="password"` -- dots for masking, matching existing API key input pattern |

## Layout Contract

### Sidebar Drawer Ordering

The Bitbucket Connection section is placed BEFORE the Provider section in the sidebar drawer:

```
sidebar-drawer
  sidebar-drawer-inner
    [NEW] Bitbucket Connection (collapsible, default: collapsed)
    Provider + API Key (collapsible, existing)
    Target Languages (collapsible, existing)
    Action / Translate button (existing)
```

**Rationale:** Bitbucket connection is a one-time setup. Once connected, users rarely revisit it. Placing it first but collapsed keeps it discoverable without cluttering the primary translation workflow.

### Bitbucket Section Internal Layout

```
.card.bitbucket-section
  .card-title-collapsible        (flex row: title + chevron)
  .collapsible-body
    .bb-field-group              (Proxy URL: label + input)
    .bb-field-group              (Workspace: label + input)
    .bb-field-group              (Repository: label + input)
    .bb-field-group              (Branch: label + input, default "main")
    .bb-field-group              (Access Token: label + password input)
    .bb-disclaimer               (muted text disclaimer)
    .bb-actions                  (flex row: badge + test button)
```

### Field Group Styling

Each `.bb-field-group` follows this pattern:

```css
.bb-field-group {
  margin-bottom: 0.75rem;  /* 12px, matches existing .provider-row spacing */
}
.bb-field-group label {
  display: block;
  font-size: 0.75rem;      /* 12px, matches .card-title */
  font-weight: 600;
  color: var(--text-muted);
  margin-bottom: 0.25rem;  /* 4px */
  text-transform: uppercase;
  letter-spacing: 0.025em;
}
.bb-field-group input {
  width: 100%;
  padding: 0.5rem 0.75rem;  /* 8px 12px, matches existing select/input */
  border: 1px solid var(--border);
  border-radius: var(--radius-sm);
  font-size: 0.875rem;      /* 14px */
  background: var(--card);
  color: var(--text);
  outline: none;
  transition: border-color var(--transition-base), box-shadow var(--transition-base);
}
.bb-field-group input:focus {
  border-color: var(--primary-border);
  box-shadow: var(--focus-ring);
}
```

## Accessibility

| Concern | Implementation |
|---------|---------------|
| Keyboard navigation | All inputs and buttons are native HTML elements (tab-navigable by default) |
| Focus visibility | `var(--focus-ring)` on all interactive elements (existing pattern) |
| Screen reader | Labels use `for` attribute linking to input `id`; section uses `role="region"` with `aria-label` |
| Color contrast | All text/background combinations meet WCAG AA (inherited from existing design system) |
| Token masking | `type="password"` on token input prevents shoulder-surfing |
