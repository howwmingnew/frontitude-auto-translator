<objective>
Create the core Bitbucket code search pipeline as a pure data layer module (`js/search.js`) that searches for translation key usage in WPF source code (.xaml / .cs) via the Bitbucket Cloud Code Search API. This module handles concurrency limiting (max 3 parallel), in-memory caching, .xaml-first-then-.cs fallback, LocExtension pattern detection, result aggregation, rate limit handling, and progress callbacks.

Purpose: This is the engine that powers context-aware translation -- knowing WHERE a key is used in the UI code lets the AI produce better translations in Phase 3.
Output: `js/search.js` module on App namespace + updated `js/state.js` with contextResults field.

Create `js/search.js` as a new IIFE module following the project's established pattern. This is the largest file in the plan -- it contains the complete search pipeline.

Structure the file with these sections, in order:

**1. IIFE wrapper and module-level variables:**
```javascript
(function () {
  'use strict';
  var App = window.App;
  var contextCache = new Map();
```

**2. Concurrency limiter factory** (`createConcurrencyLimiter(maxConcurrent)`):
- Returns a function `limit(fn)` that queues async tasks and runs at most `maxConcurrent` in parallel
- Implementation: queue array, running counter, `tryNext()` drains queue
- Use the exact pattern from 02-RESEARCH.md Pattern 1

**3. Search API call function** (`doSearch(query)`):
- Reads state fresh via `App.getState()` each call (avoids stale state per Pitfall 5)
- Constructs URL: `s.proxyUrl.replace(/\/+$/, '') + '/bitbucket/2.0/workspaces/' + encodeURIComponent(s.bitbucketWorkspace) + '/search/code'`
- Query param: `?search_query=` + `encodeURIComponent(query)` + `&pagelen=20`
- Auth header: `{ 'Authorization': 'Bearer ' + s.bitbucketToken }`
- Uses `App.fetchWithTimeout(url, opts)` for the HTTP call
- On 429 status: throw `{ type: 'rate_limit', retryAfter: parseInt(res.headers.get('Retry-After')) || 60 }`
- On non-ok status: throw `new Error('Search API error: HTTP ' + res.status)`
- On success: `return (await res.json()).values || []`

**4. Exponential backoff retry wrapper** (`withRetry(fn, maxRetries, baseDelay)`):
- maxRetries defaults to 3, baseDelay defaults to 1000ms
- On `rate_limit` type errors: wait `err.retryAfter * 1000` or `baseDelay * Math.pow(2, attempt)`, then retry
- On other errors: re-throw immediately
- Use the pattern from 02-RESEARCH.md

**5. Parse search results** (`parseSearchResults(apiValues)`):
- Takes the `values` array from Bitbucket API response
- For each item: extract `item.file.path` (default 'unknown')
- For each `content_match` in `item.content_matches`: for each `lineObj` in `cm.lines`:
  - Reconstruct full line text: `lineObj.segments.map(seg => seg.text).join('')`
  - Record line number: `lineObj.line`
- Returns array of `{ file, line, snippet }` where snippet is all matched lines joined with `\n`

**6. LocExtension pattern detection** (`detectLocExtension(snippet)`):
- Checks if a code snippet contains WPF LocExtension patterns
- Regex: `/\{(?:lex|loc):(?:Loc|Translate)\s+Key\s*=/i`
- Returns boolean
- This is used for enriching result metadata, not for search queries

**7. Build key result** (`buildKeyResult(key, searchResults)`):
- Calls `parseSearchResults(searchResults)`
- Enriches each match with `isLocExtension: detectLocExtension(match.snippet)`
- Returns: `{ key: key, matches: parsedResults, searchedAt: Date.now() }`

**8. Query length safety check** (`buildSafeQuery(key, ext)`):
- ext is 'xaml' or 'cs'
- Full query pattern: `'"' + key + '" ext:' + ext + ' repo:' + App.getState().bitbucketRepo`
- If query length > 250 chars (Bitbucket limit): drop `ext:` and `repo:` modifiers, use just `'"' + key + '"'`
- Returns the query string

**9. Single-key search** (`searchKeyContext(key)`):
- Check cache first: `if (contextCache.has(key)) return contextCache.get(key)`
- Build .xaml query via `buildSafeQuery(key, 'xaml')`
- Call `await withRetry(function() { return doSearch(query); })`
- If results empty, build .cs query via `buildSafeQuery(key, 'cs')`, search again
- Build result via `buildKeyResult(key, results)`
- Store in cache: `contextCache.set(key, result)`
- Update state immutably: `var newMap = new Map(App.getState().contextResults); newMap.set(key, result); App.setState({ contextResults: newMap });`
- Return result

**10. Untranslated key filter** (`filterUntranslatedKeys(allKeys, targetLangs, jsonData, isRetranslate)`):
- If isRetranslate is true: return all keys (per CONTEXT.md locked decision)
- Otherwise: return keys where for ANY target language, the key is missing or empty string
- "Missing" = `jsonData[lang][key] === undefined || jsonData[lang][key] === ''`
- Keys with cached results are NOT filtered out here (cache check happens in searchKeyContext)

**11. Batch search** (`searchBatchContext(keys, onProgress)`):
- Create concurrency limiter: `var limit = createConcurrencyLimiter(3)`
- Track progress: `var completed = 0; var total = keys.length;`
- Fire initial progress: `onProgress({ completed: 0, total: total, currentKey: '', status: 'searching' })`
- Update state: `App.setState({ searchProgress: { completed: 0, total: total, currentKey: '', status: 'searching' } })`
- Map all keys through limiter:
  ```javascript
  var promises = keys.map(function(key) {
    return limit(function() {
      return searchKeyContext(key).then(function(result) {
        completed++;
        var progress = { completed: completed, total: total, currentKey: key, status: 'searching' };
        onProgress(progress);
        App.setState({ searchProgress: progress });
        return result;
      }).catch(function(err) {
        completed++;
        var progress = { completed: completed, total: total, currentKey: key, status: 'searching' };
        onProgress(progress);
        App.setState({ searchProgress: progress });
        // Skip failed keys, continue batch (per CONTEXT.md: on other errors, skip key and continue)
        return null;
      });
    });
  });
  ```
- `var results = await Promise.all(promises);`
- Final progress: `App.setState({ searchProgress: { completed: total, total: total, currentKey: '', status: 'idle' } });`
- Return `results.filter(Boolean)` (remove nulls from failed keys)

**12. Cache management** (`clearContextCache()`):
- `contextCache = new Map();`
- `App.setState({ contextResults: new Map() });`

**13. Cache getter** (`getContextCache()`):
- Returns `contextCache` (for testing/debugging)

**14. Init function** (`initSearch()`):
- No-op for now (Phase 2 does not wire triggers -- that is Plan 02 for UI)
- Just a placeholder for the App.initSearch call

**15. Exports on App namespace:**
```javascript

App.filterUntranslatedKeys = filterUntranslatedKeys;
App.clearContextCache = clearContextCache;
App.getContextCache = getContextCache;
App.detectLocExtension = detectLocExtension;
App.initSearch = initSearch;
```

**16. Close IIFE:**
```javascript
})();
```

- Use `var` not `let/const` (ES5 style with async/await exception per CLAUDE.md)
- IIFE wrapper: `(function () { 'use strict'; ... })();`
- All state reads via `App.getState()` (never cache state in long-lived variable)
- Immutable state updates: create new Map, then setState
- No ES Module syntax
